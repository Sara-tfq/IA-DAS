Guide Complet - Pipeline de Génération d'Ontologies IA-DAS

Objectif : Ce guide détaille la procédure complète pour générer automatiquement les ontologies IA-DAS à partir des données sources.

Prérequis

Logiciels requis

  • Python 3.8+ installé sur votre machine
  • Java 8+ (JDK ou JRE) - nécessaire pour RMLMapper
  • RMLMapper JAR (détaillé ci-dessous)

Vérification de Java

Testez que Java est installé avec la commande :

java -version
Si Java n'est pas installé :
  • Windows : Téléchargez OpenJDK ou Oracle JDK depuis le site officiel
  • macOS : brew install openjdk
  • Linux : sudo apt install default-jdk

Téléchargement du RMLMapper

Télécharger le JAR RMLMapper version 7.3.3-r374
Lien : RMLMapper GitHub Release
Placer le fichier JAR
Copier rmlmapper-7.3.3-r374-all.jar dans le dossier pipeline-ontologie/
Note : Vous pouvez utiliser une version différente du RMLMapper, mais vous devrez adapter les noms de fichiers dans le code Python correspondant.

Données Sources

Fichiers CSV requis

Placez les 3 fichiers CSV sources dans le dossier data-csv/ avec les noms exacts suivants :

  • IA-DAS-Data.csv - Données principales des analyses
  • Sport-Hierarchy.csv - Hiérarchie des sports
  • Class-Hierarchy-V1.csv - Hiérarchie des classes de variables
Format important : Les fichiers doivent être encodés en UTF-8, c'est à dire lors du téléchargement depuis excel pour choisissez csv-UTF-8. Si vous modifiez les noms, adaptez le fichier pipeline_complet_ia_das.py en conséquence.

Exécution de la Pipeline

Méthode recommandée : Exécution automatique

Lancer la pipeline complète
Ouvrez un terminal dans le dossier pipeline-ontologie/ et exécutez : 
python pipeline_complet_ia_das.py

Processus automatisé

Le script pipeline_complet_ia_das.py exécute séquentiellement :

  1. Nettoyage des fichiers CSV
    • Suppression des lignes et colonnes vides
    • Standardisation de l'encodage
  2. Adaptation pour RMLMapper
    • Création d'identifiants uniques pour les données principales
    • Conversion des séparateurs de ; vers , (exigence RMLMapper)
  3. Génération des ontologies
    • Utilisation des mappings RML du dossier mapping/
    • Création de 3 ontologies distinctes
  4. Post-traitement
    • Nettoyage des URIs (suppression des %20)
    • Optimisation du format Turtle

Structure des Résultats

Ontologies générées

Résultats finaux dans le dossier resultats/ :
  • ia-das-ontology-clean.ttl - Ontologie principale des données d'analyse
  • sport-hierarchy-simple-clean.ttl - Hiérarchie des sports simplifiée
  • variable-hierarchy-clean.ttl - Classification des variables d'analyse

Fichiers de Mappings

Les règles de transformation sont définies dans le dossier mapping/ :

  • mapping-ontology.ttl - Mapping des données principales
  • mapping-sport.ttl - Mapping de la hiérarchie des sports
  • mapping-hierarchy.ttl - Mapping de la hiérarchie des variables

Dépannage

Problèmes courants

  • Erreur Java : Vérifiez que Java est installé et accessible
  • Fichier JAR introuvable : Vérifiez l'emplacement et le nom du fichier RMLMapper
  • Erreur d'encodage : Assurez-vous que vos CSV sont en UTF-8
  • Données manquantes : Vérifiez la présence et le nommage des fichiers sources

Scripts de vérification

  • verification_donnees.py - Vérifie l'intégrité des données sources
  • verification_statistiques.py - Génère des statistiques sur les résultats
