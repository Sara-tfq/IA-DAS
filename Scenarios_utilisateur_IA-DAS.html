IA-DAS - Scénarios d'utilisation détaillés

Document : Guide de scénarios pratiques

Auteur : Sara TAOUFIQ

Équipe : Stephanie MERIAUX, Meggy HAYOTTE, Molka TOUNSI, Amandine DAUBRESSE

Public : Utilisateurs souhaitant découvrir les fonctionnalités de l'interface IA-DAS

Pour réaliser ces scénarios , il faut d'abord lire la partie "Présentation générale" du guide utilisateur.

Scénario 1 : Recherche d'analyses sur la dépression chez les femmes

Contexte : En tant qu'utilisatrice passionnée par la recherche, je maîtrise déjà certaines notions de base. Je souhaite explorer des analyses spécifiques concernant la dépression chez les femmes. Je me dirige donc vers la page d'interaction personnalisée pour effectuer une recherche ciblée.

Parcours étape par étape :

Étape 1 : Accès à la page d'interaction personnalisée
Action :
Je clique sur le bouton "Interaction personnalisée"
Résultat : Le site commence à charger les données nécessaires. Cette opération prend quelques secondes pendant lesquelles un indicateur de chargement est visible. Une fois terminé, le système m'informe que la page est prête à l'utilisation.
Étape 2 : Sélection du critère démographique
Action :
Je clique sur le menu déroulant de sélection du sexe
Résultat : Trois options me sont proposées : "Femme", "Homme", "Mixte"
Action :
Je sélectionne "Femme"
Étape 3 : Exploration des catégories de variables
Action :
Je clique sur la sélection des catégories de variables
Résultat : Six choix différents me sont proposés pour catégoriser ma recherche
Décision :
Aucune catégorie ne correspond à mes besoins actuels, je ne sélectionne donc rien et laisse ce champ vide
Étape 4 : Saisie du facteur d'intérêt
Action :
Je commence à saisir le mot "Dépression" dans le champ de facteur
Résultat : Un système d'autocomplétion intelligent s'active et affiche une liste de suggestions prédéfinies correspondant à ma saisie, facilitant ainsi la sélection du terme exact
Étape 5 : Lancement de la recherche initiale
Action :
Je clique sur le bouton "Rechercher"
Résultat : Un popup de chargement s'affiche pour indiquer que la requête est en cours de traitement. Après quelques instants, les résultats apparaissent sous forme de tableau structuré.
Résultats obtenus : 12 analyses traitant spécifiquement des femmes avec un facteur de dépression
Étape 6 : Basculement vers la visualisation graphique
Action :
Je clique sur l'onglet "Graphique"
Résultat : Le tableau se transforme en représentation graphique avec un code couleur distinctif. Chaque nœud et connexion est coloré selon sa catégorie, facilitant l'interprétation visuelle des relations entre les données.
Étape 7 : Exploration interactive du graphique
Action :
Je survole différents nœuds avec ma souris
Résultat : Des informations préliminaires s'affichent au survol, donnant un aperçu rapide du contenu de chaque nœud sans nécessiter de clic
Étape 8 : Exploration de la hiérarchie conceptuelle
Action :
Je maintiens un clic droit prolongé sur un nœud de variable (par exemple "Anxiété")
Résultat : Un panneau hiérarchique s'ouvre sur le côté, révélant :
  • L'arborescence conceptuelle complète de cette variable
  • Les concepts parents (catégories supérieures)
  • Les concepts enfants (sous-catégories)
  • Les relations sémantiques entre les différents niveaux
Navigation hiérarchique :
Je peux cliquer sur n'importe quel élément de la hiérarchie pour naviguer dans l'ontologie et comprendre la structure conceptuelle des données
Fermeture :
Je clique à côté du panneau ou utilise le bouton X pour le fermer
Étape 9 : Analyse approfondie d'un nœud spécifique
Action :
Je double-clique sur le nœud "Dépression"
Résultat : Un panneau latéral s'ouvre, présentant toutes les analyses liées à ce nœud. Le panneau contient une liste détaillée avec :
  • Le titre de chaque analyse formaté selon les normes APA
  • L'identifiant unique de l'analyse
  • Les variables indépendantes (VI) et dépendantes (VD) étudiées
  • Le type de résultat de la relation observée
Étape 9 : Consultation détaillée d'une analyse
Action :
Je clique sur une analyse spécifique dans la liste
Résultat : Un nouveau panneau s'ouvre, affichant l'intégralité des informations concernant cette analyse : métadonnées complètes, méthodologie, résultats, références bibliographiques, etc.
Navigation :
J'utilise le bouton "Retour à la liste" pour revenir au panneau précédent
Étape 10 : Export des résultats en PDF
Action :
Je clique sur le bouton "Exporter PDF"
Résultat : La liste complète des analyses est automatiquement téléchargée sur mon ordinateur sous format PDF, prête pour archivage ou partage
Fermeture :
Je clique sur le bouton X pour fermer le panneau
Étape 11 : Export des données en Excel
Action :
Je clique sur le bouton "Export Excel"
Résultat : Un fichier Excel (.xlsx) contenant toutes les données du tableau est automatiquement téléchargé sur mon ordinateur. Le fichier inclut :
  • Les métadonnées de chaque analyse
  • Les variables indépendantes et dépendantes
  • Les informations démographiques
  • Une mise en forme professionnelle pour faciliter l'analyse
Étape 12 : Consultation de la requête SPARQL générée
Action :
Je clique sur l'onglet "SPARQL"
Résultat : La vue bascule vers l'affichage technique montrant :
  • La requête SPARQL complète qui a été générée automatiquement
  • Les résultats bruts au format JSON
  • La structure des données telle qu'elle est interrogée dans l'ontologie
Avantage :
Cette vue permet aux utilisateurs techniques de comprendre comment leurs critères ont été traduits en requête sémantique et d'analyser la structure de l'ontologie
Étape 13 : Affinement par critère d'âge
Action :
Je clique sur la sélection de catégorie d'âge
Résultat : Plusieurs options s'affichent, incluant une option "Personnalisée"
Action :
Je sélectionne "Personnalisée"
Résultat : Trois champs de saisie apparaissent pour définir l'âge minimum, maximum et moyen
Saisie :
Je renseigne la valeur 21 dans le champ approprié
Étape 14 : Recherche affinée et export final
Contexte :
La requête traite maintenant spécifiquement les femmes d'environ 21 ans avec le facteur Dépression
Action :
Je clique à nouveau sur "Rechercher"
Résultat : Les résultats se sont affinés : il ne reste plus que 2 analyses correspondant à ces critères plus précis
Export :
Je clique sur "Exporter en PNG"
Export final : Une image haute définition du graphique est téléchargée sur mon ordinateur, incluant le logo du projet en qualité optimale (qualité 6)

Scénario 2 : Utilisation de la page de compétences (Interaction prédéfinie)

Contexte : Je suis un professionnel de santé qui souhaite obtenir rapidement des réponses à des questions de compétences spécifiques sans avoir à construire moi-même des requêtes SPARQL. J'utilise donc la page d'interaction prédéfinie qui propose des questions préformulées.

Parcours étape par étape :

Étape 1 : Accès à la page de compétences
Action :
Je clique sur le bouton "Interaction prédéfinie"
Résultat : La page se charge et affiche une interface accordéon avec plusieurs sections repliables
Étape 2 : Exploration de l'interface accordéon
Action :
Je parcours les différentes sections de l'accordéon
Résultat : Chaque section contient des débuts de questions prédéfinies organisés par thématiques (compétences, relations, facteurs démographiques, etc.)
Étape 3 : Construction guidée d'une question
Action :
Je sélectionne le début de question "Quels sont les ACADS"
Résultat : Un menu déroulant s'active pour choisir la suite de la question
Action :
Je sélectionne dans le menu déroulant "dont le résultat de relation est ambigu"
Résultat : La question complète se forme automatiquement : "Quels sont les ACADS dont le résultat de relation est ambigu ?"
Étape 4 : Exécution de la requête prédéfinie
Action :
Je clique sur "Exécuter la requête"
Résultat : Les résultats s'affichent avec les mêmes options de visualisation que la page personnalisée : tableau, graphique, et vue SPARQL
Étape 5 : Analyse des résultats
Action :
J'explore les résultats en basculant entre les différents modes de visualisation
Résultat : Je bénéficie des mêmes fonctionnalités interactives qu'en mode personnalisé : double-clic sur les nœuds, export PDF, survol informatif, etc.

Scénario 3 : Page de modification (Administration)

Contexte : Je suis membre de l'équipe de recherche et j'ai besoin de mettre à jour l'ontologie en ajoutant de nouvelles analyses issues d'une étude récente. J'accède donc à la page de modification avec mes identifiants administrateur.
Attention : Ce scénario nécessite des droits administrateur et un mot de passe d'accès.

Parcours étape par étape :

Étape 1 : Accès sécurisé
Action :
Je clique sur le bouton "Mise à jour de l'ontologie"
Résultat : Un modal d'authentification s'ouvre, demandant le mot de passe administrateur
Action :
Je saisis mon mot de passe administrateur
Résultat : Après vérification, j'accède à l'interface d'administration
Étape 2 : Vue d'ensemble de l'interface d'administration
Interface disponible : Deux sections principales sont visibles :
  • Section "Modifier les analyses existantes"
  • Section "Ajouter/Supprimer des analyses"
Étape 3 : Ajout d'une nouvelle analyse
Action :
Je clique sur "Ajouter une analyse"
Résultat : Un formulaire détaillé s'ouvre avec les champs suivants :
  • Titre de l'analyse (format APA)
  • Variables indépendantes (VI)
  • Variables dépendantes (VD)
  • Type de résultat de relation
  • Métadonnées (auteurs, année, journal, etc.)
  • Critères démographiques (âge, sexe, population)
Étape 4 : Saisie des informations
Action :
Je remplis tous les champs requis avec les données de la nouvelle étude
Validation :
Je clique sur "Valider et intégrer"
Résultat : Le système vérifie la cohérence des données et confirme l'intégration réussie dans l'ontologie. Un ID unique est automatiquement généré pour la nouvelle analyse.
Étape 5 : Modification d'une analyse existante
Action :
Je clique sur "Modifier" et recherche une analyse spécifique par son ID
Résultat : L'analyse s'affiche dans un formulaire pré-rempli, permettant la modification de tous ses attributs
Modification :
Je corrige une erreur dans les métadonnées et sauvegarde
Résultat : La modification est automatiquement sauvegardée dans la base Fuseki et un historique de la modification est conservé
Étape 6 : Vérification et validation
Action :
Je teste la nouvelle analyse ajoutée en retournant sur la page d'interaction personnalisée
Résultat : La nouvelle analyse apparaît bien dans les résultats de recherche correspondants, confirmant que l'intégration s'est déroulée correctement
Sécurité : Toutes les actions d'administration sont tracées et un backup automatique de l'ontologie est créé avant chaque modification importante.
